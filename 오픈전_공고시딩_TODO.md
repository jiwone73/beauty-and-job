# 뷰티워크 오픈 전 · 공고 시딩 To-Do (1단계)

> 목표: **오픈 전에 사이트를 "살아있게"** 만든다. 텅 빈 사이트에 사람을 부르지 않기 위해, 실제 같은 구인 공고를 미리 채워둔다.
> 원칙: **넓고 얇게 X → 좁고 진하게 O.** 한 권역 + 1~2개 업종에 집중해서 그 안을 꽉 채운다.
> 권장 목표: 공고 **50~100개**, 한 권역(예: 홍대·마포) 집중.

---

## Step 1. 타겟 좁히기  _(0.5일)_

- [ ] 집중 권역 1곳 선정 (예: 홍대·마포 / 강남 / 분당)
- [ ] 집중 업종 1~2개 선정 (예: 속눈썹, 네일 — 매장 수 많고 이직 잦은 업종 우선)
- [ ] 목표 공고 수 설정 (예: 60개)
- [ ] 목표 매장 수 설정 (예: 40곳)

> 팁: 이직·구인이 잦은 **속눈썹·네일·왁싱**이 공고 채우기 쉽고 구직자도 많다. 미용실(헤어)은 대형·프랜차이즈가 많아 초기엔 후순위.

---

## Step 2. 공고·매장 소스 수집  _(2~3일)_

**어디서 모으나**

- **네이버 지도 / 스마트플레이스** — 매장 기본정보(상호·주소·업종·톡톡 여부). 매장 풀 확보용.
- **당근마켓 동네 구인구직** — 소규모 매장 실제 구인글 1순위.
- **알바천국 · 알바몬** — 뷰티/미용 카테고리 실제 구인글.
- **네이버 카페(미용 구인구직 카페)** — 원장·사장 구인글.
- **인스타그램 해시태그** — #네일구인 #속눈썹구인 #왁싱구인 #미용구인.

**할 일**

- [ ] 수집용 스프레드시트 생성 (아래 열 구성)
      `상호 / 업종 / 모집직무 / 지역 / 주소 / 급여 / 근무형태 / 연락처 / 톡톡여부 / 출처URL / 위도 / 경도 / 상태`
- [ ] 각 소스에서 대상 수집 → 시트에 정리
- [ ] 상호·주소 기준 **중복 제거**

**⚠️ 법적 주의 (중요)**

- 타 플랫폼 공고 **문구를 그대로 복사하지 말 것** (저작권·부정경쟁 소지). 공개된 **사실 정보(상호·위치·직무·급여)** 만 취하고 **문구는 새로 작성**.
- "대신 등록"은 가능하면 **사장님 동의(톡톡)** 를 받아두면 가장 안전.
- 스크래핑보다 **공식 API 사용**(아래 자동화 참고)이 약관·법적으로 안전.

---

## Step 3. 공고 문안 작성  _(1~2일)_

- [ ] 공고 템플릿 확정 (제목 / 모집직무 / 자격·경력 / 급여 / 근무조건 / 매장소개)
- [ ] 시트 데이터로 각 공고 상세 문구 작성 (사실정보 기반, 재작성)
- [ ] 썸네일 확보 (매장 공개 이미지 활용 여부 판단 / 없으면 업종별 기본 이미지)

---

## Step 4. 위경도(지오코딩) 채우기

- [ ] 시트의 각 주소 → **위도·경도 변환**
- [ ] 시트 위/경도 열 채우기

> 이유: "내 주변 채용" 지도 기능은 공고에 **위경도가 있어야** 마커가 뜬다. 지금 샘플 공고들이 위경도 비어 있으면 지도에 안 나옴. → 자동화 2번으로 일괄 처리.

---

## Step 5. 뷰티워크에 일괄 등록

- [ ] 등록 방식 결정: 관리자 UI 수동 vs **CSV 일괄 import** (자동화 4번 권장)
- [ ] 공고 정상 노출 확인 (리스트 / 검색 / 지도 마커)
- [ ] 품질 점검 (빈 필드·깨진 이미지·위치 오류 없는지)

---

## Step 6. 오픈 직전 체크

- [ ] 공고 목표 수(예: 60개) 이상 등록 완료
- [ ] 권역·업종 분포 확인 (한쪽에 안 쏠렸는지)
- [ ] **지원 플로우 최종 테스트** (구직자 지원 → 기업/관리자 알림·이력서 확인)
- [ ] 모바일·PC 표시 최종 확인

---

## 🤖 자동화 포인트 (솔로 개발자 기준)

수작업으로 100개를 관리자 화면에 하나씩 넣으면 며칠이 날아간다. 아래를 스크립트로 만들면 **수집→등록**이 반나절로 줄어든다.

1. **매장 리스트 수집 자동화**
   - 네이버 **지역검색 API** / 카카오 **로컬(키워드) API** 로 `지역+업종` 검색 → 상호·주소·카테고리를 CSV로 추출.
   - (네이버/카카오 화면 스크래핑은 약관 위반 소지 → **공식 API 사용**.)

2. **지오코딩 배치**
   - 주소 → 위경도 자동 변환 스크립트. **Kakao Geocoder**(이미 지도 기능에 사용 중)를 재활용해 CSV 전체를 한 번에 처리.

3. **공고 문안 자동 생성**
   - 수집한 사실 데이터(상호·직무·급여) + 템플릿/AI로 공고 상세를 자동 초안 생성 → 검수만.

4. **CSV 일괄 등록 (가장 큰 시간 절약)**
   - 완성된 CSV → **Supabase 일괄 insert 스크립트**. 관리자 UI 수동 입력을 아예 대체.

5. **중복·상태 관리**
   - 시트 수식/스크립트로 중복 제거 + `상태`(수집/작성완료/등록완료/동의받음) 관리.

---

## 요약 흐름

**타겟 좁히기 → (API로) 매장 수집 → 공고 문안 작성 → 지오코딩 배치 → CSV 일괄 등록 → 품질·지원 플로우 점검 → 오픈**
